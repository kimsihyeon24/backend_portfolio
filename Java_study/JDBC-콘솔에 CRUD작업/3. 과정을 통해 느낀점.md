느낀점

객체를 나누어서 작업을 함으로써 유지보수성 높인 코드 구현

JDBC를 사용하려면 connector라이브러리 필요함.

하지만 gradle과 maven사용하면 이들이 쉽게 관리해준다고함

지금 하나의 클라이언트만 서버에 연결해서 작업하느라 어려움을 느끼지는 못했지만

서버와 클라이언트를 연결하는 비용이 크다.

또한 쓰레드를 무한 생성하게 됨으로 서버가 죽을 가능성 있음

-> 그래서 나중에 쓰레드 풀이라는 걸 사용함(내일 프로젝트로 구현해볼것)



장점

1. 쓰레드 생성 비용 절감
2. 동시 처리 가능 작업수 절감
3. 요청 많은 상황에서 서버를 죽이지 않기 위해 사용



아직 안해봐서 잘 모르겠지만 과정은

db 연결 요청 -> 큐에 넣고 -> 쓰레드 풀에서 관리하고 -> 비지니스 로직

과정으로 구현될 것으로 예상
